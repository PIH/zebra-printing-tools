Browser → Zebra printer via Browser Print

Print labels directly from the browser to any Zebra ZPL printer the Browser Print helper can see, with no print dialog and no per-print user interaction. Two operations on the same ZPL textarea: Print the ZPL directly (vector primitives rendered by the printer firmware — sharpest, smallest payload) or Generate a PDF preview via the shim's zpl2pdf endpoint, then print the rasterised result via the helper's PDF→ZPL conversion. The PDF preview is also useful as a layout sanity-check before printing — saves wasted labels while iterating on formatting. There's also a skip-preview shortcut for printing existing PDFs (e.g. server-generated from another system).

ZPL

This is the ZPL that will be sent to the printer. The default below assumes a 3" × 2" label (sized to the printer's detected DPI); edit ^PW (print width, in dots) and ^LL (label length, in dots) if your loaded media is different. Anything else can be edited freely — pre-generated ZPL from the OpenMRS module or another system can be pasted here directly.

Generated PDF

Rendered server-side by zpl2pdf via the shim's POST /zpl-to-pdf endpoint. Updates whenever you click Generate PDF above (auto-fired once on printer-select). Useful for previewing labels without wasting media while iterating on the ZPL.

Print uploaded PDF

If you already have a PDF, skip the preview and print it directly via the helper conversion (Zebra.Printer.getConvertedResource). Same conversion pipeline that the Generated PDF section uses to print the ZPL→PDF round-trip.

Diagnostics & raw commands

Send a Zebra command directly to the selected printer and view the raw response. These calls bypass the SDK request queue and are independent of the print pathways above — useful for setup, troubleshooting, and exploring what the firmware actually exposes. Control characters in the response are decoded as <STX>, <ETX>, etc.; a hex dump is included below the decoded text.
