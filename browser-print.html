Browser → Zebra printer via Browser Print

Two pathways, side-by-side, same printer / dpi / inputs: (1) Direct ZPL built from primitives, (2) PDF converted to ZPL by the Browser Print helper (Zebra's licensed converter, or the included Linux shim). Built on Zebra's Browser Print SDK 3.1.250 — BrowserPrint for transport, Zebra.Printer for parsed status. Resolution and label dimensions auto-detect from the selected printer; override any field to print at a different size.

2. PDF → ZPL — converted by the Browser Print helper needs Zebra PDF feature key (or the bundled shim)

Calls Zebra.Printer.getConvertedResource(pdfBlob, {toFormat:'zpl', featureKey}), which asks the local helper to convert the PDF to printer-language using Zebra's own halftoning / dithering. Without a valid feature key the official helper rejects the request; the included browser-print-shim.py ignores the key and converts via Ghostscript + Floyd-Steinberg + ZPL ^GF compression.

Diagnostics & raw commands

Send a Zebra command directly to the selected printer and view the raw response. These calls bypass the SDK request queue and are independent of the print pathways above — useful for setup, troubleshooting, and exploring what the firmware actually exposes. Control characters in the response are decoded as <STX>, <ETX>, etc.; a hex dump is included below the decoded text.
